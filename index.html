GB v2 Block Format

Every GenerateBlocks element has 3 attributes:

  1. styles — camelCase JSON (e.g. backgroundColor)
  2. css — compiled CSS string the server renders
  3. HTML tag — carries class only, never inline style=

Native WP blocks (headings, paragraphs, buttons) do use inline styles.

Carousel Rules
  • Carousel items need dual class: gb-carousel-item gb-carousel-item-{id}
  • Use \u002d\u002d encoding for -- in JSON (CSS vars)
  • data-autoplay-enabled="true" on carousel div
  • Smooth scroll: add transition-timing-function: linear via Additional CSS
  • Editor settings after paste: Free Mode ON, Delay 1, Speed 5000, Loop ON
Section Patterns
  • Section → Container → Content (every section)
  • Section = <section> with bg + padding
  • Container = <div> with max-width + auto margins
  • Use clamp() for responsive padding
  • Eyebrow: 11px, uppercase, letter-spacing 2.5px, primary color
Heading Hierarchy
  • H1 — one per page, hero only
  • H2 — section headings
  • H3 — card/subsection titles
  • Subtitles in hero = styled <p>, not H2
Color System
  • Body text on white: body_text
  • Body text on cards: card_text
  • Light section bg: section_light
  • Dark section bg: section_dark
  • Keep one dark bg value — don't mix shades
JSON ↔ HTML Parity Rule

Critical: Every CSS property in a native WP block's HTML style="" must also exist in the block comment JSON style object, and vice versa.

  • WP generates expected HTML from the JSON comment, then compares to saved HTML
  • Any mismatch → "Block contains unexpected or invalid content" recovery error
  • Common trap: font-style:normal in HTML but missing from JSON typography — remove it (it's the browser default)
  • Applies to: wp:heading, wp:paragraph, wp:button, wp:cover, etc.
  • Does not apply to GB elements (they use styles JSON + css string)
GB Pro Carousel Blocks

The Reviews Carousel and Logo Carousel use GB Pro carousel blocks:

  • generateblocks-pro/carousel — wrapper with data-slides-per-view, data-space-between
  • generateblocks-pro/carousel-items — track container
  • generateblocks-pro/carousel-item — individual slide
  • generateblocks-pro/carousel-pagination — bullet dots nav
  • generateblocks-pro/carousel-control — prev/next arrow buttons
  • CSS custom properties use \u002d\u002d encoding for -- in JSON

After paste: Open carousel settings → enable Autoplay, set Loop ON, adjust delay/speed as needed.

After Pasting Code
  1. Open WP Code Editor → paste → switch to Visual
  2. Check for "Attempt Recovery" errors — if found, verify JSON ↔ HTML parity (see above)
  3. For carousels: configure autoplay / loop / speed in the carousel sidebar
  4. For smooth scroll: add CSS override in Appearance → Additional CSS
